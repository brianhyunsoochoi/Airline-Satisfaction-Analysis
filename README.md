# FDA Project Repository

This repository contains coursework-style machine learning experiments and utilities for airline passenger satisfaction prediction and data visualization.

## Project Structure

- `A3/` — Main Assignment 3 machine learning pipelines and experiments.
  - `data/` — Input datasets used by training/inference scripts.
  - `results/` — Output plots generated by model evaluation scripts.
  - `run.py` — Multi-model training/evaluation pipeline.
  - `version1.py` — Cross-validation model comparison and plot generation.
  - `testboost.py` — XGBoost train/evaluate/predict pipeline.
  - `testrf.py` — RandomForest train/evaluate/predict pipeline.
  - `testgridcv.py` — RandomForest hyperparameter search with GridSearchCV.
  - `teststacks2.py`, `teststacks3.py` — Stacking ensemble experiments.
  - `voting.py` — Soft-voting ensemble experiment.
  - `featurecorrelation.py` — Correlation analysis and heatmap generation.
- `visualization.py` — Generates univariate and multivariate EDA charts from spreadsheet/csv data.
- `analysis.py`, `excel_writer.py`, `1a2.py` — Additional analysis/helper scripts.
- `FDA A3/` — Collected image outputs and figures.

## Requirements

The scripts rely on common Python ML/data libraries:

- Python 3.9+
- `pandas`
- `numpy`
- `scikit-learn`
- `matplotlib`
- `seaborn`
- `xgboost` (for XGBoost scripts)
- `joblib`

Install dependencies:

```bash
pip install pandas numpy scikit-learn matplotlib seaborn xgboost joblib openpyxl
```

## Data Files

Expected A3 dataset files:

- `A3/data/train_subset.csv`
- `A3/data/test_kaggle_features.csv`
- `A3/data/test_subset_for_kaggle_sample_submission.csv` (optional, depending on script)

Some scripts assume execution from inside the `A3/` directory because of relative paths like `data/train_subset.csv`.

## How to Run

### 1) Main multi-model pipeline

```bash
cd A3
python run.py
```

### 2) XGBoost pipeline with submission output

```bash
cd A3
python testboost.py
```

### 3) RandomForest pipeline with submission output

```bash
cd A3
python testrf.py
```

### 4) Grid search tuning

```bash
cd A3
python testgridcv.py
```

### 5) Stacking and voting experiments

```bash
cd A3
python teststacks2.py
python teststacks3.py
python voting.py
```

### 6) Correlation analysis

```bash
cd A3
python featurecorrelation.py
```

### 7) Visualization script (root directory)

```bash
python visualization.py
```

By default, `visualization.py` attempts to load:

- `32130_AT2_25716438.xlsx` (preferred)
- `32130_AT2_25716438.csv` (fallback)

Generated figures are written to `figs_1A/`.

## Outputs

Depending on the script, outputs include:

- Console metrics (Accuracy, F1, AUC, classification reports)
- Kaggle-style submission CSV files (e.g., `submission.csv`)
- Model comparison charts and confusion matrix/ROC images
- Feature importance summaries and saved model artifacts

## Notes

- Many scripts are experiment-focused and may overlap in preprocessing logic.
- Ensure you run scripts from the expected working directory to avoid file path errors.
- If you want a unified, production-style pipeline, consider consolidating shared preprocessing into a single reusable module.
